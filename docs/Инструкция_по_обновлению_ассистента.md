# Инструкция по обновлению ассистента (ответы на отзывы клиентов и аналитика)

Учебный проект. Обновления делаются вручную: 
- в интерфейсе ChatGPT
- в Playground
- в Google Docs
- в Google Sheets

---

## 1. Где что хранится

| Артефакт                     | Где править                                                                            |
|------------------------------|----------------------------------------------------------------------------------------|
| Системный промпт             | `data/Промпт_и_сценарии.md` → ссылка на Google Doc                                     |
| База знаний: типовые ответы  | `data/База_знаний.md` → ссылка на Google Sheet → лист "Типовые фразы и готовые ответы" | 
| База знаний: примеры отчётов | `data/База_знаний.md` → ссылка на Google Sheet → листы "Пример отчета: ..."            |
| Тестовые отзывы клиентов     | `data/Тестовые_отзывы_клиентов.md` → ссылка на Google Sheet с отзывами клиентов для тестирования    |
| Данная инструкцмя            | `docs/Инструкция_по_обновлению_ассистента.md`                                          |

---

## 2. Как обновить системный промпт

1. Откройте актуальный текст системного сообщения в `Google Doc`.  
2. Внесите изменения (новые сценарии, уточнение тона, новые ограничения).  
3. **Версионируйте:** в шапке документа или в комментарии укажите дату и кратко «что изменилось».  
4. В ChatGPT: *Настройки → Personalization → Custom instructions* — вставьте обновлённый блок целиком.  
5. В Playground: замените содержимое поля **System** и сохраните пресет (если используете).

**Совет:** не раздувайте промпт дублированием примеров; лучше ссылаться на то, что пользователь «прикрепит» фрагмент из базы знаний.

### OpenAI Playground: как именно подать базу знаний

**Вариант 1 — Google Таблица → копипаст**

1. Откройте [platform.openai.com/playground](https://platform.openai.com/playground), режим **Chat**.  
2. В поле **System** вставьте системный промпт из Google Doc, который находится по ссылке в `Системный_промпт_ассистент_отзывы.md`.
3. Выделите нужные столбцы из `Google Sheet`, которая находится по ссылке в `База_знаний.md`, **вместе с строкой заголовков**, скопируйте (**Ctrl+C**).  
4. В поле **User** первым сообщением вставьте текст вида:

   `Ниже база знаний в формате CSV (заголовок и строки). Используй её при классификации отзывов и при составлении ответов. Дальше я буду присылать только отзывы.`

Таблица в буфере часто вставляется с табуляцией — для модели это нормально; при желании уточните в тексте: «данные разделены табуляцией».

5. Ниже — **скопируйте содержимое** базы знаний из `Google Sheet`. Отправьте сообщение.  
6. Следующими сообщениями **User** присылайте только отзывы и задачи («классифицируй», «предложи ответ» и т.д.). Модель опирается на уже отправленную таблицу, пока диалог укладывается в лимит контекста.

**Вариант 2 — использовать Vector Store**

1. Импортируйте базу знаний и примеры запросов клиентов из Google Sheets в PDF-файлы.
2. В OpenAI Playground добавьте в векторное хранилище импортированные PDF-файлы: 
- Выберите **Tools → Add → File Search**
- В открывшемся окне загрузите PDF-файлы и задайте имя векторного хранилища
- Дополнительные параметры хранилища оставьте по умолчанию
- Нажмите **Attach** — создастся векторное хранилище с заданным именем

## 3. Как обновить базу знаний (Google Sheet)

1. **Новая строка** = новая типовая ситуация или уточнение шаблона.  
2. Поля Google Sheet:  
   - `id` — уникальный номер;  
   - `Тип сценария` — тип (латиница/транслит удобны для фильтра);  
   - `Типовый фразы` — ключевые корни/слова для сопоставления (можно через `|`);  
   - `Шаблон ответа` — готовый каркас ответа;  
   - `Пометка для сотрудников` — подсказка оператору.
3. **Проверка:** обновить Vector Store или попросите ассистента в тестовом диалоге: «Вот строка из базы знаний и отзыв — сопоставь и адаптируй ответ.»

---

## 4. Частота пересмотра (рекомендация для учебного проекта)

- Промпт: при смене «личности» бренда, тона или новых жёстких ограничений компании.  
- Таблица ответов: при появлении **новых частых формулировок** в отзывах.  
- Примеры отчётов: обновлять при смене KPI или структуры отчёта у заказчика-кейса.

---

## 5. Контроль качества перед сдачей / демо

- [ ] Ограничения «не публиковать / не удалять» присутствуют в системном промпте.  
- [ ] В примерах нет реальных ФИО, телефонов, адресов (только вымышленные).  
- [ ] В Google Sheet нет пустых обязательных полей в новых строках.  
- [ ] Успешное прохождение тестов в соответствии с разделом "Как протестировать ассистента" в [readme.md](README.md).
